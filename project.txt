TABULAR ML PIPELINE - PORTOFOLIO PROJECT (Updated: December 31, 2025)
====================================================================

Deskripsi Proyek
----------------
Proyek ini adalah end-to-end configurable ML pipeline untuk data tabular, dirancang khusus agar mudah digunakan di Kaggle Playground Series atau kompetisi tabular lainnya.

Pipeline ini mengutamakan:
- Modularitas (setiap tahap terpisah)
- Config-driven (semua pengaturan via file YAML, cukup copy & edit config baru untuk eksperimen berbeda)
- Reproducibility (seed, logging, folder output terstruktur)
- Best practices software engineering (src layout, unit tests, clean structure)
- Performa komputasi tinggi (optimasi loading, preprocessing, dan training)
- Kemudahan maintenance dan skalabilitas untuk masa depan

Cocok sebagai portofolio hybrid Software Engineer + AI/ML karena menunjukkan kemampuan membangun sistem ML yang rapi, testable, efisien, dan production-ready.

Optimasi Performa Komputasi (Fokus Utama Update)
-----------------------------------------------
Pipeline ini telah dioptimasi secara khusus untuk kecepatan eksekusi dan efisiensi memori:

1. Format Data: Parquet (snappy compression)
   - Load time: 5–20x lebih cepat dibanding CSV
   - Ukuran file: 3–10x lebih kecil
   - Memori: columnar storage + predicate pushdown (hanya baca kolom dibutuhkan)

2. Data Loading
   - pd.read_parquet otomatis digunakan jika ekstensi .parquet
   - Selective column loading (opsional di config nanti)

3. Preprocessing & Memory
   - Downcast numeric types (float64 → float32, int64 → int32)
   - OneHotEncoder dengan sparse=True (hemat memori drastis jika categorical banyak)
   - Vectorized operations only (no slow apply/lambda loops)

4. Feature Engineering
   - Semua operasi vectorized (Pandas/NumPy)
   - Enable/disable per fitur via config untuk eksperimen cepat

5. Training
   - Prioritas model cepat: LightGBM > CatBoost > XGBoost
   - Parallel processing (n_jobs=-1 di CV dan model)
   - Parameter efisien default (max_bin, subsample, early_stopping)

6. Caching & Reusability
   - Data processed disimpan di data/processed/ (Parquet)
   - Preprocessor/model bisa di-cache dengan joblib (opsional fase lanjutan)

Cara Menggunakan (Diperbarui)
----------------------------
1. Download dataset Kaggle → taruh di data/raw/ sebagai CSV
2. Jalankan sekali: python scripts/convert_to_parquet.py
   → Otomatis konversi semua CSV menjadi Parquet di data/processed/
3. Update config/*.yaml:
   - Ganti train_path dan test_path ke .parquet
4. Jalankan: python main.py --config default (atau nama config lain)
5. Hasil muncul di experiments/<nama_experiment>/
